**Step 2: Rewrite the file validation in the upload_and_optimize endpoint**

Replace the existing extension and size validation (from `filename = file.filename or ""` through the size check) with enhanced validation. The validation must run BEFORE `tempfile.NamedTemporaryFile` (extension and content-type checks) and before CSV parsing (size check is fine where it is since we need to read content).

```python
# --- File validation (BEFORE any processing) ---
filename = file.filename or ""
ext = pathlib.Path(filename).suffix.lower()

# Extension check with descriptive error
if ext not in ALLOWED_EXTENSIONS:
    raise HTTPException(
        status_code=400,
        detail=f"Unsupported file type ({ext or 'none'}). Accepted: {', '.join(sorted(ALLOWED_EXTENSIONS))}",
    )

# Content-type check (some browsers send odd types, so be lenient but log)
content_type = file.content_type or ""
if content_type and content_type not in ALLOWED_CONTENT_TYPES:
    raise HTTPException(
        status_code=400,
        detail=f"Unexpected content type ({content_type}). Upload a CSV or Excel file.",
    )

suffix = ".csv" if ext == ".csv" else ".xlsx"
tmp_path: str | None = None
preprocessed_path: str | None = None
try:
    with tempfile.NamedTemporaryFile(delete=False, suffix=suffix) as tmp:
        content = await file.read()
        # Size check with actual and max size in error message
        if len(content) > MAX_UPLOAD_SIZE_BYTES:
            size_mb = len(content) / (1024 * 1024)
            max_mb = MAX_UPLOAD_SIZE_BYTES / (1024 * 1024)
            raise HTTPException(
                status_code=413,
                detail=f"File too large ({size_mb:.1f} MB). Maximum: {max_mb:.0f} MB.",
            )
        tmp.write(content)
        tmp_path = tmp.name
```

**Key differences from the current code:**
1. Uses `pathlib.Path(filename).suffix.lower()` instead of `filename.lower().endswith(...)` — more robust, handles edge cases
2. Extension error includes the rejected extension AND the accepted list
3. Content-type validation added (new)
4. Size error includes actual file size in MB, not just the limit
5. All validation happens BEFORE `await file.read()` for extension/content-type, and BEFORE `tmp.write()` for size

In `tests/apps/kerala_delivery/api/test_api.py`, find the `rate_limited_client` fixture inside `TestRateLimiting` class (around line 1619). Add `limiter.reset()` to the teardown (after yield, before `limiter.enabled = False`):

```python
@pytest.fixture
def rate_limited_client(self, mock_session):
    """TestClient with rate limiting ENABLED."""
    from apps.kerala_delivery.api.main import app, limiter

    async def override_get_session():
        yield mock_session

    limiter.enabled = True
    app.dependency_overrides[get_session] = override_get_session
    yield TestClient(app)
    app.dependency_overrides.clear()
    # CRITICAL: reset all in-memory counters to prevent state leaking
    # to subsequent test modules. Without this, another module enabling
    # the limiter may encounter pre-existing counter state from these tests.
    limiter.reset()
    # Disable again for other test classes
    limiter.enabled = False
```

The key addition is `limiter.reset()` which clears all in-memory rate limit counters. This prevents the following scenario: TestRateLimiting runs, bumps counters to near the limit, a later test module re-enables the limiter, and immediately gets 429 because the old counters are still active.
