Uses DaisyUI alert + collapse components with tw- prefix.

Output: Updated types.ts, api.ts, and UploadRoutes.tsx with import summary section.

1. Add a new workflow state "partial-success" to handle the zero-geocoded case distinctly:

Per CONTEXT.md decision, there are three UI scenarios when upload completes:
- **All succeed (zero failures):** Green summary bar + route cards below
- **Partial success (some failures):** Amber summary bar with expandable failures + route cards below
- **Zero success (all fail):** Amber/red summary bar with expandable failures + NO route cards + message "No orders could be geocoded -- check addresses below"

The simplest approach: keep the existing "success" workflow state but render different content based on `uploadResult` field values. Add a check for `uploadResult.geocoded === 0 && uploadResult.failures.length > 0` to show zero-success state.

2. In `UploadRoutes.tsx`, create an `ImportSummary` inline component (or a section within the success render). Place it BETWEEN the upload area and the route cards (per user decision).

The ImportSummary section should:

a) **Summary counts bar** (always visible):
   - Use DaisyUI `tw-alert` component
   - When `failures.length === 0`: `tw-alert tw-alert-success` with text "All {total_rows} orders geocoded successfully"
   - When failures exist: `tw-alert tw-alert-warning` with three stat chips:
     - "{orders_assigned} routed" (bold count)
     - "{orders_unassigned} unassigned" (if > 0)
     - "{failed_geocoding + failed_validation} failed" (bold count, red text)

b) **Expandable failure detail table** (collapsed by default, only when failures exist):
   - Use DaisyUI `tw-collapse tw-collapse-arrow tw-bg-base-200`
   - Title: "{failures.length} failed row(s) -- click to expand"
   - Content: a `tw-table tw-table-sm` with columns: Row, Address, Reason, Stage
   - Each failure row shows: `f.row_number`, `f.address_snippet || '--'`, `f.reason`, `f.stage`

c) **Warnings section** (if warnings array is non-empty):
   - Separate from failures -- use DaisyUI `tw-collapse` with a lighter style
   - Title: "{warnings.length} warning(s) -- defaults applied"
   - Same table format as failures

d) **Zero-success message** (when `geocoded === 0` and failures exist):
   - Below the summary bar: a prominent message "No orders could be geocoded -- check addresses below"
   - Do NOT render route cards section
   - Still show the expandable failure table so staff can see what went wrong
