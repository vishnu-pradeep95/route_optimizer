<task type="auto">
  <name>Task 2: Create API endpoint E2E tests (TEST-01)</name>
  <files>e2e/api.spec.ts</files>
  <action>
Create `e2e/api.spec.ts` with full JSON schema validation for all API endpoints per user decision. Use Playwright's `request` fixture (APIRequestContext) -- no browser needed.

Structure:
- `test.describe.configure({ mode: 'serial' })` -- tests run in order
- `test.beforeAll` -- call `validateApiKey()`, then `uploadTestCSV()` to ensure routes exist for subsequent GET tests. Store the upload response for assertions.

Test groups (use `test.describe` for organization):

**Health & Config:**
- `GET /health` -- expect 200, body matches `{ status: 'ok', service: 'kerala-lpg-optimizer', version: expect.any(String), environment: expect.any(String) }`
- `GET /api/config` -- expect 200, body has `depot_lat`, `depot_lon`, `max_vehicles`, `max_weight_kg`

**Upload (already done in beforeAll, validate response):**
- Validate upload response matches `{ run_id: expect.any(Number), orders_received: expect.any(Number), orders_assigned: expect.any(Number), vehicles_used: expect.any(Number), routes: expect.any(Array) }` with `orders_assigned > 0`

**Routes:**
- `GET /api/routes` -- expect 200, body is array with length > 0, each route has `vehicle_id`, `driver_name`, `stops` (array)
- `GET /api/routes/{vehicle_id}` -- use first vehicle_id from /api/routes response, expect 200, body has `vehicle_id`, `stops` (array with length > 0), each stop has `order_id`, `address`, `latitude`, `longitude`, `sequence`, `status`
- `POST /api/routes/{vehicle_id}/stops/{order_id}/status` -- use first vehicle's first stop, POST `{ status: "delivered" }`, expect 200

**Vehicles:**
- `GET /api/vehicles` -- expect 200, body is array, each has `vehicle_id`, `driver_name`, `max_weight_kg`
- `GET /api/vehicles/{vehicle_id}` -- use first vehicle_id, expect 200
- `POST /api/vehicles` -- create test vehicle `{ vehicle_id: "TEST-E2E", driver_name: "E2E Test Driver", max_weight_kg: 400 }`, expect 200 or 201
- `PUT /api/vehicles/TEST-E2E` -- update driver_name, expect 200
- `DELETE /api/vehicles/TEST-E2E` -- expect 200

**Optimization Runs:**
- `GET /api/runs` -- expect 200, body is array with length > 0 (from the upload in beforeAll)
- `GET /api/runs/{run_id}/routes` -- use run_id from upload response, expect 200, body is array

**Telemetry:**
- `POST /api/telemetry` -- send `{ vehicle_id, latitude, longitude, timestamp, event_type: "location_update" }`, expect 200
- `GET /api/telemetry/fleet` -- expect 200
- `GET /api/telemetry/{vehicle_id}` -- expect 200

**QR Sheet:**
- `GET /api/qr-sheet` -- expect 200, content-type contains 'text/html'

**Error cases:**
- `POST /api/upload-orders` without X-API-Key header (send empty header) -- expect 401 or 403
- `GET /api/routes/NONEXISTENT-VEHICLE` -- expect 404 or empty response

Use `expect(body).toMatchObject({...})` with `expect.any()` matchers for schema validation per user decision. Do NOT use CSS selectors or hard-coded waits. Use web-first assertions where applicable.
