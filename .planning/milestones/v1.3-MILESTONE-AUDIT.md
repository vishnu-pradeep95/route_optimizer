### Deployment Blocker: osrm-init exit 127 (User-reported)

**Reported by user on fresh machine deployment (DESKTOP-D77351B).**

`docker-compose.yml` line 71 uses `image: osrm/osrm-backend:latest` with `entrypoint: ["/bin/bash", "-c"]`. The `latest` tag now pulls an Alpine-based image that has no `/bin/bash`, causing exit 127 ("command not found"). The production compose (`docker-compose.prod.yml`) pins `v5.27.1` which still has bash.

**Root cause:** Unpinned `latest` tag + bash-specific entrypoint on an image that dropped bash.

**Fix options:**
1. Pin version to `v5.27.1` (matching docker-compose.prod.yml)
2. Change entrypoint to `["/bin/sh", "-c"]` (the script only uses POSIX constructs)
3. Both (pin + use sh for resilience)

**Impact:** Blocks ALL fresh deployments. bootstrap.sh -> install.sh -> `docker compose up -d --build` triggers osrm-init which fails immediately. start.sh also triggers it via `docker compose up -d`.
