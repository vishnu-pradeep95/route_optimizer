**Step 1: Query the production database for confidence distribution.**

Run a SQL query against the PostgreSQL database (via `docker compose exec -T postgres psql -U postgres -d routing_opt`) to get:

```sql
-- Overall row count and source distribution
SELECT source, COUNT(*) as count,
       ROUND(AVG(confidence)::numeric, 3) as avg_confidence,
       ROUND(MIN(confidence)::numeric, 3) as min_confidence,
       ROUND(MAX(confidence)::numeric, 3) as max_confidence
FROM geocode_cache
GROUP BY source
ORDER BY count DESC;

-- Confidence tier distribution (matching _confidence_tier logic)
SELECT
  CASE
    WHEN confidence >= 0.90 THEN 'rooftop (>=0.90)'
    WHEN confidence >= 0.70 THEN 'interpolated (>=0.70)'
    WHEN confidence >= 0.50 THEN 'geometric_center (>=0.50)'
    ELSE 'approximate (<0.50)'
  END as tier,
  COUNT(*) as count,
  ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER (), 1) as pct
FROM geocode_cache
GROUP BY tier
ORDER BY MIN(confidence) DESC;

-- Exact confidence value distribution (to see actual clustering)
SELECT confidence, source, COUNT(*) as count
FROM geocode_cache
GROUP BY confidence, source
ORDER BY confidence DESC, count DESC;
```

If the database has zero rows (no production data yet), document that finding and state that thresholds remain as-is since there is no data to contradict them.
