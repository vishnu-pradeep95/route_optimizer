Three changes to index.html, all in the script section:

**1. Config fetch on app init (PWA-01):**
Add a `fetchAppConfig()` async function near the top of the script (after the auth helpers, before SW registration ~line 919). This function:
- Fetches `${API_BASE}/api/config`
- On success, updates the Call Office FAB href: `document.getElementById('call-office-fab').href = 'tel:' + data.office_phone_number`
- Caches the config in localStorage using `saveToStorage(STORAGE_KEYS.APP_CONFIG, data)` so it works offline
- On fetch failure, falls back to cached config from localStorage
- If neither fetch nor cache available, hide the FAB entirely (set display to 'none') rather than showing a broken phone link
- Add `APP_CONFIG: 'lpg_app_config'` to the STORAGE_KEYS object

Call `fetchAppConfig()` early in the app init flow -- at the bottom of the script where the app bootstraps (around line 1695-1700), before or alongside the route restoration logic.

Also remove the hardcoded `tel:+919876543210` from the HTML `<a>` tag at line 843. Change to `href="#"` as a safe default (will be overwritten by config fetch).

Remove the TODO comment at line 842 (`<!-- TODO: Replace with actual Vatakara HPCL office number -->`).

**2. GPS watchPosition leak fix (PWA-02):**
- Add a `let gpsWatchId = null;` variable alongside the existing `telemetryIntervalId` declaration (around line 885)
- In `startTelemetry()` (line 1522), capture the return value: `gpsWatchId = navigator.geolocation.watchPosition(...)`
- In `stopTelemetry()` (line 1542), add GPS cleanup BEFORE the interval cleanup:
  ```javascript
  if (gpsWatchId !== null) {
  ```

