    1. **Create auto_create_drivers_from_csv helper function** in `apps/kerala_delivery/api/main.py`:

       ```python
       async def auto_create_drivers_from_csv(
           session: AsyncSession,
           preprocessed_df: pd.DataFrame,
       ) -> dict:
           """Auto-create drivers from CSV DeliveryMan column.

           For each unique delivery_man in the DataFrame:
           1. Fuzzy-match against ALL existing drivers (including deactivated)
           2. If match found and active: record as "existing"
           3. If match found and inactive: reactivate, record as "reactivated"
           4. If no match: create new driver, record as "new"

           CRITICAL: Do NOT merge intra-CSV name variations. Process each unique
           name independently against the EXISTING driver database at the start.
           New drivers created from this CSV are NOT used as match targets for
           other names in the same CSV.

           Returns dict with new_drivers, matched_drivers, reactivated_drivers, existing_drivers.
           """
       ```

       Implementation details:
       - Extract unique delivery_man values from preprocessed_df
       - Take a snapshot of existing drivers BEFORE processing (so intra-CSV names don't match each other)
       - For each unique name:
         - Compute fuzzy scores against the snapshot (not against newly created drivers)
         - If best match score >= DRIVER_MATCH_THRESHOLD:
           - If matched driver is active: add to existing_drivers list
           - If matched driver is inactive: call reactivate_driver(), add to reactivated_drivers with name
         - If no match: call create_driver(name), add to new_drivers
       - Return summary dict
