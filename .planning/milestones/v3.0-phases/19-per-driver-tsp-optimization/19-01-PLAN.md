Create `core/optimizer/tsp_orchestrator.py` with these functions:

1. `group_orders_by_driver(orders: list[Order], order_driver_map: dict[str, str]) -> dict[str, list[Order]]`
   - Takes final geocoded orders list and a pre-built order_id->driver_name map
   - Groups orders by driver name. Orders not in the map are excluded (logged as warning).
   - Returns dict[driver_name, list[Order]]

2. `optimize_per_driver(orders_by_driver: dict[str, list[Order]], driver_uuids: dict[str, uuid.UUID], depot: Location, optimizer: VroomAdapter) -> tuple[RouteAssignment, list[str]]`
   - For each driver: create Vehicle(vehicle_id=driver_name, driver_name=driver_name, max_weight_kg=99999.0, max_items=999, depot=depot)
   - Call optimizer.optimize(driver_orders, [vehicle]) per driver
   - Wrap each call in try/except -- on failure, add driver's order IDs to unassigned, add warning string (per research discretion: partial results)
   - Skip drivers with 0 orders
   - Merge all results: combine routes into one list, combine unassigned, sum optimization_time_ms
   - Set route_id = f"R-{assignment_id}-{driver_name}" for each route
   - Set driver_id on each Route if available from driver_uuids (add driver_id to Route model if not present -- but Route model doesn't have it; store it on RouteDB during persistence instead)
   - Returns (RouteAssignment, warnings: list[str])

3. `validate_no_overlap(assignment: RouteAssignment) -> list[str]`
   - Check every order_id across all routes. If any appears twice, return error message: "Order {order_id} assigned to both {driver_a} and {driver_b}"
   - Uses route.vehicle_id as driver identifier (contains driver name per decision)

4. `detect_geographic_anomalies(assignment: RouteAssignment, overlap_threshold: float = 0.30) -> list[str]`
   - For each route with 3+ stops, compute convex hull using shapely.geometry.MultiPoint
   - For each pair of driver hulls, check intersection area ratio
   - If max(intersection/hull_a, intersection/hull_b) > threshold, add warning: "Geographic overlap: {driver_a} and {driver_b} have {pct}% delivery area overlap -- consider reassigning orders in CDCMS"
   - Use (longitude, latitude) order for Shapely points (Shapely uses x,y = lon,lat)

Write tests FIRST in `tests/core/optimizer/test_tsp_orchestrator.py`. Mock `VroomAdapter.optimize` using unittest.mock. Create test fixtures with sample Order objects having known locations. Use Shapely test points that produce known overlap percentages.
