    **Gap 1 — Geocoding filter (BLOCKER):**

    In `upload_and_optimize()` in main.py, the driver selection filter is at lines 1848-1867 (after geocoding). Move it BEFORE the geocoding loop (before line 1622). The logic:

    1. After `orders = import_result.orders` (line 1439) and after `area_name_map` is built (line 1468-1475), but BEFORE the geocoding section (line 1583), add the driver filter:

    ```python
    # Phase 17 gap closure: Filter orders by selected drivers BEFORE geocoding.
    # Previously this filter was after geocoding (wasting API calls on deselected
    # drivers' orders). Now only selected drivers' orders are geocoded.
    if selected_driver_list is not None and is_cdcms and not preprocessed_df.empty:
        order_driver_map: dict[str, str] = {}
        if "delivery_man" in preprocessed_df.columns:
            for _, row in preprocessed_df.iterrows():
                oid = str(row.get("order_id", "")).strip()
                dm = str(row.get("delivery_man", "")).strip()
                if oid and dm:
                    order_driver_map[oid] = dm

        selected_set = set(selected_driver_list)
        before_filter = len(orders)
        orders = [
            o for o in orders
            if order_driver_map.get(o.order_id, "") in selected_set
        ]
        # Also filter area_name_map to only selected orders
        area_name_map = {k: v for k, v in area_name_map.items() if k in {o.order_id for o in orders}}
        logger.info(
            "Driver selection: %d of %d orders selected for geocoding (%d drivers selected)",
            len(orders), before_filter, len(selected_set),
        )
    ```

    2. Remove the DUPLICATE filter block at lines 1848-1867 (the old post-geocoding filter). Replace it with just `orders_for_optimization = geocoded_orders` (the variable assignment the optimizer expects). The comment and the entire filter block are no longer needed since filtering happened earlier.

    **Gap 2 — Allocation Pending filter (MAJOR):**

    In `cdcms_preprocessor.py`, add a placeholder filter step AFTER the status filter (line 238) and BEFORE the delivery_man filter (line 247). Define a set of known CDCMS placeholder names:

    ```python
    # Known CDCMS placeholder values in DeliveryMan column.
    # These are not real drivers — they indicate unassigned orders.
    PLACEHOLDER_DRIVER_NAMES = {"ALLOCATION PENDING", ""}
    ```

    Add a new Step 3b after Step 3 (status filter):

    ```python
    # Step 3b: Filter out placeholder driver names (not real drivers)
    if CDCMS_COL_DELIVERY_MAN in df.columns:
        before = len(df)
        df = df[
            ~df[CDCMS_COL_DELIVERY_MAN].str.strip().str.upper().isin(PLACEHOLDER_DRIVER_NAMES)
        ]
        filtered_count = before - len(df)
        if filtered_count > 0:
            logger.info(
                "Filtered %d placeholder driver entries (Allocation Pending, blank)",
                filtered_count,
            )
    ```

    This filters at the preprocessor level, which means both parse_upload AND upload_and_optimize endpoints benefit — "Allocation Pending" orders never appear in preview or get geocoded.
