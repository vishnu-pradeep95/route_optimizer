Add the license renewal mechanism and a shared get_license_info() accessor.

Purpose: Enable customers to renew their license by dropping a renewal.key file and restarting the API, without needing a new fingerprint exchange. The get_license_info() accessor is the shared foundation for the expiry header and health endpoint enrichment in plan 02.

Output: Working renewal flow (generate key with --renew, drop renewal.key, restart API), get_license_info() accessor, updated docker-compose.prod.yml with renewal.key mount.

  <behavior>
    - enforce() checks for renewal.key before calling validate_license()
    - If renewal.key exists and is valid (VALID or GRACE status), it becomes the primary license
    - After successful renewal: license.key is replaced with renewal key content, renewal.key is deleted
    - If renewal.key is invalid (wrong fingerprint, expired beyond grace, tampered), falls through to normal license.key validation with a warning log
    - If renewal.key does not exist, normal validation proceeds unchanged
    - Renewal key file paths checked: ["renewal.key", "/app/renewal.key"]
    - File replacement is best-effort (log warning if write fails, don't crash)
  </behavior>
  <action>
    1. In enforcement.py, add `_try_load_renewal_key()` helper function before `enforce()`:
       ```python
       def _try_load_renewal_key() -> tuple[LicenseInfo | None, str | None]:
           """Try to load and validate renewal.key.
           Returns (LicenseInfo, key_content) if valid, (None, None) otherwise."""
           for path in ["renewal.key", "/app/renewal.key"]:
               try:
                   with open(path, "r") as f:
                       key = f.read().strip()
                   if key:
                       info = validate_license(key)
                       if info.status in (LicenseStatus.VALID, LicenseStatus.GRACE):
                           return info, key
                       else:
                           logger.warning("renewal.key present but invalid: %s", info.message)
               except FileNotFoundError:
                   continue
           return None, None
       ```

    2. Add `_handle_post_renewal(key_content: str)` helper for file replacement:
       - Write key_content to license.key (or /app/license.key if that path exists)
       - Delete renewal.key (or /app/renewal.key)
       - Both operations wrapped in try/except with warning logs on failure
       - This is best-effort: if the Docker volume is read-only, the warning is logged but the API still runs with the renewed license in memory

    3. In `enforce()`, add renewal check as Step 0 BEFORE Step 1 (validate_license):
       ```python
       # Step 0: Check for renewal.key
       renewal_info, renewal_key = _try_load_renewal_key()
       if renewal_info:
           logger.info(
               "License renewed via renewal.key -- customer=%s, new expiry=%s",
               renewal_info.customer_id,
               renewal_info.expires_at.strftime("%Y-%m-%d"),
           )
           license_info = renewal_info
           _handle_post_renewal(renewal_key)
       else:
           # Step 1: Normal validation
           license_info = validate_license()
       ```
       CRITICAL: renewal must happen BEFORE set_license_state() because the one-way state guard blocks INVALID->VALID upgrades. The renewal key becomes the initial license_info, and set_license_state() is called once with the renewed state.

       Add a comment: "# Renewal key -- drop alongside license.key, restart API to renew"

         - test_enforce_fallback_on_invalid_renewal_key: renewal.key with wrong fingerprint, falls through to license.key

         - test_post_renewal_replaces_license_key: verify license.key written and renewal.key deleted
         - test_post_renewal_handles_readonly_gracefully: simulate OSError on write, verify warning logged but no crash
