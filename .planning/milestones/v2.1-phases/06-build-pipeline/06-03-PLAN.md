**File 1: infra/cython_build.py**
```python
"""Cython build configuration for licensing module compilation.

Used inside infra/Dockerfile.build to compile license_manager.py to a
platform-specific .so extension. Run with: python cython_build.py build_ext --inplace

Scope: core/licensing/license_manager.py ONLY.
__init__.py is kept as a plain .py stub (see 06-RESEARCH.md Pitfall 1).
"""
from setuptools import setup, Extension
from Cython.Build import cythonize

extensions = [
    Extension(
        "core.licensing.license_manager",
        ["core/licensing/license_manager.py"],
        extra_compile_args=["-O2"],
    ),
]

setup(
    ext_modules=cythonize(
        extensions,
        compiler_directives={
            "language_level": "3",
            "embedsignature": False,
        },
    ),
)
```

Key points per BLD-03:
- `-O2` optimization flag via `extra_compile_args`
- `embedsignature=False` so `.__doc__` on compiled functions returns None (harder to reverse)
- `language_level: "3"` for Python 3 semantics
- Only `license_manager.py` is compiled (NOT `__init__.py` per Pitfall 1)

**File 2: infra/Dockerfile.build**
```dockerfile
# Cython build image for licensing module compilation.
# Shares python:3.12-slim base with runtime Dockerfile for ABI compatibility.
# Build tools (gcc, python3-dev, Cython) are NOT in the runtime image.
FROM python:3.12-slim AS cython-builder

RUN apt-get update && \
    apt-get install -y --no-install-recommends gcc python3-dev && \
    rm -rf /var/lib/apt/lists/*

RUN pip install --no-cache-dir Cython==3.2.4

WORKDIR /build

# Copy only what's needed for compilation
COPY core/ core/
COPY infra/cython_build.py .

# Compile license_manager.py to .so
RUN python cython_build.py build_ext --inplace

# Validate the .so imports correctly inside this container
RUN python -c "from core.licensing.license_manager import get_machine_fingerprint; print('Import validation OK')"
```

**Step 2: STRIP DEV-MODE from staged main.py (NEW -- after rsync, before ENVIRONMENT validation)**
After rsync copies the source tree to $STAGE, the staged main.py still contains the `_is_dev_mode = os.environ.get("ENVIRONMENT") == "development"` gate and all the conditional blocks that use it. In the distributed build, production is the ONLY mode -- there is no ENVIRONMENT toggle. Use sed to strip the dev-mode gate from the staged copy:

```bash
# ═══════════════════════════════════════════════════════════════════════════
# Strip dev-mode gate from staged main.py (distribution is production-only)
# ═══════════════════════════════════════════════════════════════════════════

header "Stripping dev-mode gate from staged main.py"

STAGED_MAIN="$STAGE/apps/kerala_delivery/api/main.py"

# Replace the _is_dev_mode check with hardcoded False
# After 06-01 refactor, main.py has:
#   _is_dev_mode = os.environ.get("ENVIRONMENT") == "development"
# Replace with:
#   _is_dev_mode = False
sed -i 's|_is_dev_mode = os.environ.get("ENVIRONMENT") == "development"|_is_dev_mode = False|' "$STAGED_MAIN"

# Similarly replace the lifespan dev check:
#   _lifespan_is_dev = os.environ.get("ENVIRONMENT") == "development"
# Replace with:
#   _lifespan_is_dev = False
sed -i 's|_lifespan_is_dev = os.environ.get("ENVIRONMENT") == "development"|_lifespan_is_dev = False|' "$STAGED_MAIN"

# Verify the replacements worked (should find _is_dev_mode = False)
grep -n "_is_dev_mode = False" "$STAGED_MAIN" || { error "Failed to strip _is_dev_mode gate"; exit 1; }
grep -n "_lifespan_is_dev = False" "$STAGED_MAIN" || { error "Failed to strip _lifespan_is_dev gate"; exit 1; }

success "Dev-mode gates hardcoded to False in staged main.py"
```

This approach preserves all the conditional logic (CORS, HSTS, /docs, license bypass) but since `_is_dev_mode = False` and `_lifespan_is_dev = False`, the dev branches never execute. The key outcome: `grep "ENVIRONMENT" "$STAGED_MAIN"` returns zero matches because the `os.environ.get("ENVIRONMENT")` calls have been removed from the staged copy.
