<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete Driver PWA refresh:
- WCAG AAA color system (7:1 body, 4.5:1 large text, 14px minimum, no muted tier)
- Segmented progress bar below sticky header (green/red/saffron/dim segments)
- "X of Y delivered" header stats + "Last updated" timestamp + Refresh button
- Hero card for next delivery (large address, meta, notes, Navigate/Done/Fail)
- Compact stop list for remaining stops
- Auto-advance with toast (1.5s feedback, then next stop slides to hero)
- Custom dark-themed fail modal (dialog element, reason dropdown)
- Call Office FAB (bottom-right, phone icon, tel: link)
- 60px+ touch targets on all primary action buttons
- Service worker cache bumped to v4
  </what-built>
  <how-to-verify>
**Setup:**
1. Start the API server: `cd apps/kerala_delivery && python -m uvicorn api.main:app --reload`
2. Open `http://localhost:8000/driver/` in Chrome
3. Open Chrome DevTools (F12) and switch to mobile view (e.g., Pixel 5, 393x851)

**Test PWA-01: Hero card**
4. Upload a test CSV and select a vehicle
5. Verify the FIRST pending stop appears as a large hero card at the top:
   - Label reads "NEXT DELIVERY . Stop 1 of N"
   - Address is large (~22px)
   - Shows cylinder count, weight, distance
   - Navigate button is full-width, large, saffron
   - Done + Fail buttons are side-by-side below Navigate

**Test PWA-02: Progress bar**
6. Verify segmented progress bar appears below the header
7. First segment should be saffron (current), rest dim (pending)
8. Header shows "0 of N delivered"
9. Tap "Done" on the hero card
10. Verify: green toast appears, header updates to "1 of N delivered", first progress segment turns green, next stop slides into hero position

**Test PWA-03: Refresh + timestamp**
11. Verify "Last updated: HH:MM AM/PM" shows below the progress bar
12. Verify Refresh button is visible next to the timestamp
13. Tap Refresh — timestamp should update

**Test PWA-04: Touch targets**
14. Verify Navigate button is visibly large (~66px height, full width)
15. Verify Done and Fail buttons are at least 60px height
16. Verify Call Office FAB is ~60px diameter circle
17. Verify "Yes, Failed" and "Cancel" in fail dialog are at least 60px height

**Test PWA-05: WCAG AAA contrast**
18. Verify NO light gray or "washed out" text anywhere — all text should be clearly readable
19. Verify two text tiers only: bright white (#F0EFFB) and medium (#A3A2BC)
20. Verify no text is smaller than ~14px
21. Verify saffron color only appears on large elements (buttons, progress segments, hero border) — not on small body text

**Test PWA-06: Offline**
22. In DevTools Network tab, switch to "Offline"
23. Reload the page — the route should still display from cache
24. Tap "Done" on the hero card — should work (optimistic update + offline queue)
25. Switch back online — verify no errors

**Test fail modal:**
26. Tap "Fail" on the hero card
27. Verify a dark dialog appears (NOT a browser confirm popup)
28. Verify optional reason dropdown works
29. Tap "Cancel" — dialog closes, nothing changes
30. Tap "Fail" again, then "Yes, Failed" — red toast appears, stop marked failed

**Test Call Office FAB:**
31. Verify round phone icon button in bottom-right corner
32. Tap it — should attempt to open phone dialer (or show tel: prompt in desktop browser)

**Test all-done state:**
33. Mark all remaining stops as Done or Failed
34. Verify all-done banner appears where the hero card was
35. Verify all progress bar segments are green/red (no pending)
