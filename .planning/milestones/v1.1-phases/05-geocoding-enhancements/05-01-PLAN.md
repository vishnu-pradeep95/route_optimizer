    # GEOCODING COST & DUPLICATE DETECTION

    **Step 2: Create `core/geocoding/duplicate_detector.py`.**

    Implement the full module with:

    1. `haversine_meters(lat1, lon1, lat2, lon2) -> float` — Great-circle distance using stdlib `math`. Earth radius 6,371,000 m.

    2. `_confidence_tier(confidence: float) -> str` — Maps 0.0-1.0 confidence to tier name:
       - >= 0.90 → "rooftop"
       - >= 0.70 → "interpolated"
       - >= 0.50 → "geometric_center"
       - < 0.50 → "approximate"

    3. `DuplicateCluster` dataclass with fields: `order_ids: list[str]`, `addresses: list[str]`, `max_distance_m: float`, `center_lat: float`, `center_lon: float`.

    4. `detect_duplicate_locations(orders: list[Order], thresholds: dict[str, float]) -> list[DuplicateCluster]`:
       - Filter to geocoded orders only (`o.is_geocoded and o.location`)
       - Return `[]` if fewer than 2 geocoded orders
       - Compute normalized addresses via `normalize_address()` for same-address exclusion
       - Pairwise O(n^2) comparison:
         - Skip pairs with identical normalized addresses (multi-cylinder deliveries)
         - Compute Haversine distance
         - Select threshold as `max(thresholds[tier_i], thresholds[tier_j])` — wider threshold dominates for mixed-confidence pairs
         - If distance <= threshold, union the pair
       - Use inline Union-Find (parent array with path compression) for transitive clustering
       - Extract clusters of size >= 2
       - For each cluster: compute max pairwise distance, centroid lat/lon
       - Return list of `DuplicateCluster` objects

    Use default confidence of 0.4 (approximate tier) when `geocode_confidence` is None.

    Import `normalize_address` from `core.geocoding.normalize`.
    Import `Order` from `core.models.order`.

    # Track per-order geocode source for frontend display

    if cached_geocoder:
        cache_hits = cached_geocoder.stats["hits"]
        api_calls = cached_geocoder.stats["misses"]

        cache_hits = len(geocoded_orders)

    else:
