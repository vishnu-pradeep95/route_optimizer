<task type="auto">
  <name>Task 1: Fix fetchHealth to preserve 503 per-service data</name>
  <files>apps/kerala_delivery/dashboard/src/lib/api.ts</files>
  <action>
Replace the `fetchHealth()` function (lines 257-260) with a direct `fetch()` call that always parses the JSON body regardless of HTTP status code. The `/health` endpoint intentionally returns HTTP 503 with a valid JSON body containing per-service status when the system is degraded or unhealthy. The current implementation uses `apiFetch()` which throws on non-2xx responses, causing `App.tsx` to lose the per-service breakdown.

Replace:
```typescript
/** Check if the backend API is reachable and healthy. */
export async function fetchHealth(): Promise<HealthResponse> {
  return apiFetch<HealthResponse>("/health");
}
```

With:
```typescript
/** Check if the backend API is reachable and healthy.
 *
 * Uses direct fetch instead of apiFetch because /health intentionally
 * returns 503 with valid JSON body on degraded/unhealthy state.
 * apiFetch throws on non-2xx, which would discard the per-service data.
 */
export async function fetchHealth(): Promise<HealthResponse> {
  const url = `${BASE_URL}/health`;
  const response = await fetch(url, { cache: "no-store" });
  // /health always returns valid JSON regardless of status code (200 or 503)
  return (await response.json()) as HealthResponse;
}
```

Do NOT change `apiFetch()` itself -- this fix is scoped to `fetchHealth` only. All other endpoints correctly throw on non-2xx.

Do NOT change `App.tsx` -- the existing `checkHealth` callback already handles the parsed response correctly. When `fetchHealth` returns instead of throwing, `setHealthData(res)` receives the full per-service data and `setApiHealthy(res.status === "healthy" || res.status === "ok")` correctly evaluates to `false` for degraded/unhealthy states.
