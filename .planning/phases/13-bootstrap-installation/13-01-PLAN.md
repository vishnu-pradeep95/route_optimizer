<task type="auto">
  <name>Task 1: Create scripts/bootstrap.sh with full installation pipeline</name>
  <files>scripts/bootstrap.sh</files>
  <action>
Create `scripts/bootstrap.sh` as an executable bash script (`chmod +x`). Use `set -euo pipefail`. The script has two major code paths: (A) fresh install (no Docker), and (B) resume after WSL restart or skip (Docker already present).

**Structure the script in this exact order:**

1. **Color helpers** -- Copy the exact color/helper functions from `scripts/install.sh` (lines 39-63: GREEN/YELLOW/RED/BLUE/BOLD/NC, info/success/warn/error/header). Match install.sh exactly for visual consistency.

2. **Spinner helper** -- Add a `spin_while()` function that shows a spinner while a background process runs:
   ```
   spin_while() { local pid=$1 msg="${2:-Working...}" chars='|/-\' i=0; while kill -0 "$pid" 2>/dev/null; do i=$(((i+1)%4)); printf "\r  %s %s  " "${chars:$i:1}" "$msg"; sleep 0.2; done; printf "\r%s\r" "$(printf ' %.0s' {1..60})"; wait "$pid"; return $?; }
   ```

3. **Resume check (FIRST thing after helpers)** -- Check for `.bootstrap_resume` marker file in project root. If found:
   - Remove the marker file
   - Print "Resuming installation after restart..."
   - Verify user is now in docker group: `groups | grep -q docker`. If NOT, print error "Docker group not applied. Close ALL Ubuntu windows, then reopen and re-run." and exit 1.
   - Fall through to .env generation (step 7) and install.sh delegation (step 8).

4. **Guard: WSL check** -- `grep -qi microsoft /proc/version 2>/dev/null`. If not on WSL, print: "This installer is designed for Windows Subsystem for Linux (WSL). For other Linux systems, use ./scripts/install.sh directly." Exit 1.

5. **Guard: WSL version** (INST-05) -- `uname -r | grep -qi "microsoft.*standard"`. If fails (WSL1 detected):
   ```
   error "WSL version 1 detected. WSL version 2 is required."
   echo ""
   echo "  To upgrade to WSL2:"
   echo "  1. Open PowerShell as Administrator"
   echo "  2. Run: wsl --set-version Ubuntu 2"
   echo "  3. Wait for conversion to complete"
   echo "  4. Re-open Ubuntu and run this script again"
   ```
   Exit 1.

6. **Guard: Windows filesystem** (INST-03) -- `case "$(pwd)" in /mnt/[a-z]/*)`. If matched:
   ```
   error "You are running from the Windows filesystem ($(pwd))"
   echo ""
   echo "  This causes severe performance problems with Docker."
   echo "  Please clone the project in your Linux home directory:"
   echo ""
   echo "  1. cd ~"
   echo "  2. Copy or clone the project folder to here"
   echo "  3. cd routing_opt"
   echo "  4. ./bootstrap.sh"
   ```
   Exit 1.

7. **Guard: RAM warning** (INST-04) -- Parse MemTotal from /proc/meminfo: `awk '/MemTotal/ {print int($2/1024)}' /proc/meminfo`. If < 5120 MB:
   ```
   warn "Low memory: ${TOTAL_RAM_MB} MB allocated to WSL (5120 MB recommended)"
   echo ""
   echo "  OSRM needs about 4-5 GB RAM to process Kerala map data."
   echo "  To increase WSL memory:"
   echo ""
   echo "  1. Open Notepad on Windows"
   echo "  2. Type these two lines:"
   echo "     [wsl2]"
   echo "     memory=6GB"
   echo "  3. Save as: C:\\Users\\YourName\\.wslconfig"
   echo "     (replace YourName with your Windows username)"
   echo "  4. Open PowerShell and run: wsl --shutdown"
   echo "  5. Re-open Ubuntu and try again"
   echo ""
   ```
   Continue (non-blocking per user decision). If >= 5120, print `success "Memory: ${TOTAL_RAM_MB} MB available"`.

8. **Docker check + install** (INST-01) -- Check if Docker is already installed AND user is in docker group:
   - If `command -v docker &>/dev/null && groups | grep -q docker`: print `success "Docker already installed"` and skip to .env generation (step 10).
   - If Docker is NOT installed: run the full Docker CE installation:
     a. Print `header "Step 1/4: Installing Docker CE"`
     b. Remove conflicting packages silently (docker.io, docker-doc, docker-compose, podman-docker, containerd, runc)
     c. `sudo apt-get update -qq`
     d. `sudo DEBIAN_FRONTEND=noninteractive apt-get install -y -qq ca-certificates curl gnupg`
     e. Add Docker GPG key to /etc/apt/keyrings/docker.asc
     f. Add Docker apt repository (one-line `.list` format per research recommendation)
     g. `sudo apt-get update -qq`
     h. `sudo DEBIAN_FRONTEND=noninteractive apt-get install -y -qq docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin`
     i. Use `spin_while` with background apt processes for the two slow apt-get install commands
     j. Print `success "Docker CE installed"`

9. **Docker auto-start** (INST-02) -- Configure wsl.conf for systemd:
   - Print `header "Step 2/4: Configuring Docker auto-start"`
   - Check if `systemd=true` already in /etc/wsl.conf. If yes, skip.
   - If `[boot]` section exists, insert `systemd=true` after it using sudo sed.
   - If no `[boot]` section, append `[boot]\nsystemd=true` using sudo tee -a.
   - Run `sudo systemctl enable docker.service` and `sudo systemctl enable containerd.service`
   - Print `success "Docker will start automatically on every boot"`

10. **Docker group + restart flow** -- If Docker was just installed (not the skip path):
   - `sudo usermod -aG docker $USER`
   - Write `.bootstrap_resume` marker file: `touch .bootstrap_resume`
   - Print clear restart instructions:
     ```
     header "Step 3/4: Restart required"
     echo ""
     echo "  Docker is installed! To continue, restart your terminal:"
     echo ""
     echo "  1. Close this Ubuntu window (type 'exit' or click X)"
     echo "  2. Open Ubuntu again from the Start menu"
     echo "  3. Run these commands:"
     echo "     cd $(pwd)"
     echo "     ./bootstrap.sh"
     echo ""
     echo "  The installation will resume automatically."
     echo ""
     ```
   - Exit 0.

11. **.env generation** (reached via resume path OR skip-Docker path) --
   - Print `header "Step 3/4: Setting up configuration"` (step 3 on resume path because Docker install was step 1-2 in the previous session; on skip path this is step 1/2)
   - IMPORTANT: Adjust step numbering dynamically. Use a variable `STEP=1` that increments. When Docker was already installed, total steps is 2 (config + install.sh). When resuming, total steps is 2 (config + install.sh).

   Actually, simplify: Use semantic headers instead of numbered steps when on the resume/skip path. The "Step N/4" pattern is for the fresh-install flow. For resume, use:
   ```
   header "Generating configuration..."
   ```

   Better approach: Track whether we're in fresh-install mode or resume/skip mode. In fresh-install mode, steps are 1/4 through 4/4. In resume mode, just use descriptive headers without numbers.

   - If .env already exists: `success ".env already exists -- keeping existing configuration"`
   - If .env does NOT exist:
     a. Check .env.example exists. If not: `error ".env.example not found. Installation package may be incomplete." && exit 1`
     b. `cp .env.example .env`
     c. Generate: `DB_PASS=$(tr -dc 'A-Za-z0-9' < /dev/urandom | head -c 32)` and `API_KEY_VAL=$(tr -dc 'A-Za-z0-9' < /dev/urandom | head -c 32)`
     d. `sed -i "s|^POSTGRES_PASSWORD=.*|POSTGRES_PASSWORD=$DB_PASS|" .env`
     e. `sed -i "s|^API_KEY=.*|API_KEY=$API_KEY_VAL|" .env`
     f. Preserve GOOGLE_MAPS_API_KEY from .env.example (no sed needed -- it's already copied)
     g. `success ".env created with auto-generated credentials"`

12. **Delegate to install.sh** --
   - Print `header "Starting system installation..."`
   - Print `info "This downloads Docker images and Kerala map data (~10-20 minutes first time)"`
   - `exec ./scripts/install.sh`

**Key constraints (from locked decisions):**
- ZERO interactive prompts -- never use `read` command
- Do NOT modify install.sh -- bootstrap pre-creates .env so install.sh skips its interactive creation
- Do NOT use `newgrp docker` -- use two-phase marker file restart
- Do NOT use `sudo docker compose` as workaround
- Credentials printed in install.sh's success banner (API_KEY is in .env which install.sh reads)
- Both entry points work: bootstrap.sh for fresh installs, install.sh standalone for developers

**Final consideration on step numbering:**
Use a simple approach: The fresh-install path has 4 logical phases displayed to the user:
- "Step 1/4: Installing Docker..." (only on fresh install)
- "Step 2/4: Configuring auto-start..." (only on fresh install)
- "Step 3/4: Restart required" (only on fresh install, then exit)
- After resume: "Generating configuration..." then "Starting installation..." (no step numbers, these are short)

For the skip-Docker path (Docker already installed):
- "Generating configuration..." then "Starting installation..." (no step numbers)

This avoids confusing step numbering across different code paths.
