<objective>
Fix osrm-init container exit 127 that blocks all fresh deployments by pinning the OSRM Docker image to v5.27.1 and switching entrypoints from /bin/bash to /bin/sh.

Purpose: The upstream osrm/osrm-backend:latest image migrated from Debian to Alpine, which lacks /bin/bash. This breaks the osrm-init container, which cascades to block osrm, vroom, and api services -- making the entire system unable to start on fresh machines.

Output: All 4 operational files updated with pinned image version and POSIX-compatible shell.
</objective>
