<objective>
Replace Python internals (set notation, list repr, raw API status codes, raw exception text) with plain-English "problem -- fix action" error messages across the upload validation and geocoding error flows.

Purpose: Office employees currently see Python set notation like `{'OrderNo', 'ConsumerAddress'}` and raw API codes like `Geocoding failed (ZERO_RESULTS)` when errors occur. These messages are meaningless to non-technical staff. Every error should tell them what went wrong and what to do about it.

Output: Modified `cdcms_preprocessor.py`, `csv_importer.py`, and `main.py` with humanized error messages; updated and new tests confirming the message format.
</objective>

           if isinstance(exc, ValueError):
               if "could not convert" in msg and "float" in msg:
                   return "Invalid number value -- check for letters or symbols in numeric fields"
               if "could not convert" in msg:

               return f"Invalid value -- {msg}" if len(msg) < 60 else "Invalid value in this row"
           if isinstance(exc, KeyError):
               col = msg.strip("'\"")
               return f"Missing required field '{col}' -- check your CSV has all required columns"
           if isinstance(exc, TypeError):
               return "Empty or invalid cell -- fill in required fields"
           return "Could not process this row -- check the data format"
