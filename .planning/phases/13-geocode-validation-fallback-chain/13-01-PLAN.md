  <behavior>
    ## Zone Check (GVAL-01)
    - is_in_zone(11.6244, 75.5796) returns True (depot itself, 0m)
    - is_in_zone(11.65, 75.56) returns True (within 30km)
    - is_in_zone(12.0, 75.0) returns False (>30km away)
    - is_in_zone(10.0, 76.0) returns False (far away - Kochi area)

    ## Full Validation - Direct Hit (GVAL-04)
    - validate(lat=11.65, lon=75.56, area_name="VADAKARA", geocoder=mock) returns ValidationResult(confidence=1.0, method='direct', lat=11.65, lon=75.56)
    - No calls to geocoder (in-zone on first check)

    ## Area-Name Retry (GVAL-02)
    - validate(lat=28.6, lon=77.2, area_name="EDAPPAL", geocoder=mock) where mock.geocode("{EDAPPAL}, Vatakara, Kozhikode, Kerala") returns in-zone coords:
      returns ValidationResult(confidence=0.7, method='area_retry', lat=in_zone_lat, lon=in_zone_lon)

    ## Area-Name Retry Also Out-of-Zone -> Centroid Fallback (GVAL-03)
    - validate(lat=28.6, lon=77.2, area_name="MUTTUNGAL", geocoder=mock) where area retry also out-of-zone, but "MUTTUNGAL" exists in dictionary:
      returns ValidationResult(confidence=0.3, method='centroid', lat=dict_lat, lon=dict_lon)

    ## Depot Fallback
    - validate(lat=28.6, lon=77.2, area_name="UNKNOWNPLACE", geocoder=mock) where area retry fails and area not in dictionary:
      returns ValidationResult(confidence=0.1, method='depot', lat=depot_lat, lon=depot_lon)

    ## area_name=None (Standard CSV - no CDCMS area)
    - validate(lat=28.6, lon=77.2, area_name=None, geocoder=mock) skips area retry, goes to centroid/depot:
      returns ValidationResult(method='centroid' or 'depot')

    ## Circuit Breaker (3 consecutive REQUEST_DENIED)
    - record_api_denial() x3 -> is_tripped == True
    - record_api_denial() x2 then record_api_success() then record_api_denial() -> is_tripped == False (reset on success)
    - When tripped: validate() skips geocoder calls, goes directly to centroid/depot fallback

    ## Centroid Lookup
    - get_centroid("MUTTUNGAL") returns (lat, lon) from dictionary (exact match)
    - get_centroid("muttungal") returns same (case-insensitive)
    - get_centroid("VADAKARA") returns centroid (matches primary name)
    - get_centroid("VATAKARA") returns centroid (matches alias)
    - get_centroid("NONEXISTENT") returns None

    ## Stats Tracking
    - After multiple validations, stats dict tracks: direct_count, area_retry_count, centroid_count, depot_count, circuit_breaker_trips
  </behavior>
  <implementation>
    Create `core/geocoding/validator.py` with:

    1. **ValidationResult dataclass** with fields: latitude, longitude, confidence, method, original_lat (optional), original_lon (optional).

    2. **GeocodeValidator class** with:
       - `__init__(depot_lat, depot_lon, zone_radius_m=30_000, dictionary_path=None, area_suffix=", Vatakara, Kozhikode, Kerala")` -- loads centroids from dictionary if path provided, initializes circuit breaker state and stats.
       - `is_in_zone(lat, lon) -> bool` -- uses `haversine_meters()` from `duplicate_detector.py`.
       - `get_centroid(area_name) -> tuple[float, float] | None` -- case-insensitive exact match against loaded dictionary names and aliases.
       - `validate(lat, lon, area_name, geocoder) -> ValidationResult` -- the full fallback chain:
         a. Check if (lat, lon) is in zone -> return direct (1.0)
         b. If circuit breaker tripped, skip to centroid/depot
         c. If area_name provided, retry geocode with `"{area_name}{area_suffix}"` -> if in zone, return area_retry (0.7)
         d. If area_name retry returned REQUEST_DENIED, call `record_api_denial()`
         e. Look up centroid for area_name -> if found, return centroid (0.3)
         f. Return depot fallback (0.1)
       - `record_api_denial()` -- increment consecutive counter, trip if >= 3.
       - `record_api_success()` -- reset consecutive counter.
       - `is_tripped` property.
       - `stats` dict for tracking validation outcomes.
       - `_load_centroids(path)` -- loads dictionary JSON, indexes by uppercase name and aliases.

    Key implementation notes:
    - Import `haversine_meters` from `core.geocoding.duplicate_detector` (do NOT rewrite it).
    - The `geocoder` parameter in `validate()` is a sync `Geocoder` (the GoogleGeocoder), not the CachedGeocoder. The validator calls `geocoder.geocode()` for area-name retry.
    - Cache out-of-zone Google results: YES (per research recommendation -- validator calls geocoder which may cache internally, but the validator itself does not cache).
    - Do NOT cache centroid/depot fallback results (per research anti-pattern).
    - Use flat 1.0 for all direct in-zone hits (per research recommendation -- 4-tier system, not 7-tier).
    - When area_name is None (standard CSV), skip area-name retry, try centroid lookup via AddressSplitter or go to depot.
    - For circuit breaker tripped path: try centroid for area_name, then depot. Do NOT call geocoder.
  </implementation>
