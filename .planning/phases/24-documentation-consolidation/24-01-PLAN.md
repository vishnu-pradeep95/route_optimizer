**Section 2: Grace Period Monitoring**
- Extract from `core/licensing/license_manager.py`:
  - GRACE_PERIOD_DAYS = 7 (line ~120)
  - LicenseStatus enum values: VALID, GRACE_PERIOD, EXPIRED, INVALID (lines ~75-88)
  - X-License-Warning header behavior (added to all responses during grace period)
- How to detect grace period: `curl -I http://localhost:8000/api/routes` and check for `X-License-Warning` header
- What the warning header contains (expiry date, days remaining)
- What happens when grace period ends (all endpoints return 503)

**Section 3: Renewal Process**
Step-by-step:
1. Get new machine fingerprint (may have changed if Docker container was recreated): exact command from existing Step 1
2. Send fingerprint to developer
3. Developer generates new key: exact command from existing Step 2
4. Customer updates LICENSE_KEY in .env or license.key file
5. Restart: `./scripts/deploy.sh` (production) or `docker compose restart api` (dev)
6. Verify: `curl http://localhost:8000/health` -- should respond normally, no X-License-Warning header

**Section 4: Troubleshooting License 503**
Use the "Problem -> fix action" pattern from Phase 17 / DEPLOY.md:

- **"All endpoints return 503 Service Unavailable"**
  Fix: License expired. Check X-License-Warning in recent responses. Follow renewal process above.

- **"503 with 'License not valid for this machine'"**
  Fix: Machine fingerprint changed (Docker container recreated, hostname changed). Re-run get_machine_id.py, send new fingerprint to developer for a new key.

- **"503 with 'Invalid license key format'"**
  Fix: Key corrupted or has extra whitespace. Check LICENSE_KEY value in .env -- must start with "LPG-", no trailing newlines. Or re-download the license.key file.

- **"Only /api/upload-orders fails, other endpoints work fine"**
  Fix: This is NOT a license issue. This is a Google Maps API error. See [GOOGLE-MAPS.md](GOOGLE-MAPS.md) for troubleshooting.

Add the cross-reference to GOOGLE-MAPS.md per the research recommendation to help users distinguish license 503 from geocoding errors.
