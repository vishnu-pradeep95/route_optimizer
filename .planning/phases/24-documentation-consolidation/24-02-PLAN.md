<task type="auto">
  <name>Task 2: Create GOOGLE-MAPS.md -- API key troubleshooting guide</name>
  <files>GOOGLE-MAPS.md</files>
  <action>
Create GOOGLE-MAPS.md in the project root. Plain-English for office employees -- step-by-step text with "you should see..." expected outputs. No screenshots (they go stale when Google updates UI). Follow DEPLOY.md tone and callout box pattern.

Structure:

1. **Audience callout box**: "> **Who this is for:** An employee at the office who needs to set up or fix Google Maps for address lookup (geocoding). No programming knowledge required."

2. **What Google Maps Does in This System** -- one paragraph explaining that the system uses Google Maps to convert delivery addresses to GPS coordinates (geocoding), and this only happens during CSV upload. If Google Maps is not working, uploads will fail but everything else (routes, driver app, delivery tracking) continues to work.

3. **Important: Not a License Error** -- brief note: "If ALL pages and endpoints return errors (not just upload), the problem is likely a license issue, not Google Maps. See [LICENSING.md](LICENSING.md#troubleshooting-license-503)." This is the cross-reference recommended in 24-RESEARCH.md to distinguish license 503 from geocoding errors.

4. **Setting Up a Google Maps API Key** -- step-by-step for Cloud Console:
   a. Go to https://console.cloud.google.com
   b. Create a project (or select existing)
   c. Navigate to "APIs & Services" > "Library"
   d. Search for "Geocoding API" and click "Enable" -- "You should see a green checkmark and 'API enabled' message"
   e. Go to "APIs & Services" > "Credentials"
   f. Click "Create Credentials" > "API key" -- "You should see a long string of letters and numbers"
   g. Copy the key
   h. Set up billing (required for Geocoding API): "APIs & Services" > "Billing" -- "Google provides $200/month free credit, which covers approximately 40,000 address lookups"
   i. Paste key into .env file: `GOOGLE_MAPS_API_KEY=your-key-here`
   j. Restart API: `docker compose restart api`
   k. Test: upload a CSV and check that addresses are converted to coordinates

5. **Validating Your Key** -- quick test command:
   ```bash
   curl "https://maps.googleapis.com/maps/api/geocode/json?address=Vatakara,Kerala&key=YOUR_KEY_HERE"
   ```
   "You should see a JSON response with `"status": "OK"` and latitude/longitude values."

6. **Common Errors** -- use the "Problem -> fix action" pattern from Phase 17:

   ### "REQUEST_DENIED"
   **What it means:** Google rejected the API request.
   **Common causes:** API key is missing or invalid, Geocoding API is not enabled, or the key has IP/domain restrictions blocking your server.
   **Fix:**
   - Check that GOOGLE_MAPS_API_KEY is set in your .env file (not empty, no extra spaces)
   - Go to Cloud Console > APIs & Services > Library > verify "Geocoding API" shows as enabled
   - Check Cloud Console > APIs & Services > Credentials > click your key > verify no IP restrictions are blocking the server
   - Run the validation curl command above to test directly

   ### "OVER_QUERY_LIMIT"
   **What it means:** Too many requests sent too quickly, or the monthly quota is exceeded.
   **Common causes:** Uploading very large CSV files (100+ addresses) or billing not set up.
   **Fix:**
   - Check billing is active in Cloud Console (the free $200/month credit requires a billing account)
   - For large uploads, wait a few minutes and retry -- the system caches results, so previously geocoded addresses won't count again
   - Check your usage: Cloud Console > APIs & Services > Geocoding API > Metrics

   ### "INVALID_REQUEST"
   **What it means:** The address sent to Google was empty or malformed.
   **Common causes:** Empty address fields in the CSV file.
   **Fix:**
   - Open the CSV file and check for rows with missing or empty address columns
   - See [CSV_FORMAT.md](CSV_FORMAT.md) for the required format

   ### "ZERO_RESULTS"
   **What it means:** Google could not find the address.
   **Common causes:** Address is too vague, misspelled, or does not exist.
   **Fix:**
   - Check the address in the CDCMS system -- is it complete enough for Google to find?
   - Try searching the address on https://maps.google.com directly
   - Add more detail (post office name, landmark, pin code)

7. **Still Not Working?** -- escalation: "If you've followed all the steps above and uploads still fail, contact the software developer with: (1) the exact error message, (2) a screenshot of your Cloud Console API dashboard showing the Geocoding API status."
