    from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type
    import httpx

    TRANSIENT_EXCEPTIONS = (httpx.ConnectError, httpx.TimeoutException, httpx.ConnectTimeout)

    geocoding_retry = retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=1, max=10),

        reraise=True,

    # ── Startup health gates (NON-NEGOTIABLE) ────────────────────────
    # Block until PostgreSQL, OSRM, VROOM are healthy (60s timeout).
    # Sequence: PostgreSQL first, then OSRM, then VROOM.
    # If timeout, start anyway but store degraded status for /health endpoint.
    osrm_url = os.environ.get("OSRM_URL", "http://osrm:5000")
    vroom_url = os.environ.get("VROOM_URL", "http://vroom:3000")

    logger.info("Checking service health (60s timeout)...")
    service_health = await wait_for_services(engine, osrm_url, vroom_url, timeout=60.0)
    app.state.service_health = service_health
    app.state.started_at = datetime.now(timezone.utc)

    unhealthy = [name for name, (healthy, _) in service_health.items() if not healthy]
    if unhealthy:
        logger.warning("Starting with unhealthy services: %s -- API will return 503 for affected operations", unhealthy)
    else:
        logger.info("All services healthy")

    **3. Replace the existing /health endpoint** (currently at line ~689) with enhanced version:
    ```python
    @app.get("/health")
    async def health_check():
        """Enhanced health check with per-service status.

        Returns overall status (healthy/degraded/unhealthy) and per-service
        breakdown. Used by start-daily.sh health polling and dashboard status bar.
        """
        service_health = getattr(app.state, "service_health", {})
        google_status, google_msg = check_google_api()

        services = {
            "postgresql": {
                "status": "connected" if service_health.get("postgresql", (False,))[0] else "unavailable",
                "message": service_health.get("postgresql", (False, "unknown"))[1],
            },
            "osrm": {
                "status": "available" if service_health.get("osrm", (False,))[0] else "unavailable",
                "message": service_health.get("osrm", (False, "unknown"))[1],
            },
            "vroom": {
                "status": "available" if service_health.get("vroom", (False,))[0] else "unavailable",
                "message": service_health.get("vroom", (False, "unknown"))[1],
            },
            "google_api": {
                "status": google_status,
                "message": google_msg,
            },
        }

        all_healthy = all(
            s["status"] in ("connected", "available", "configured", "not_configured")
            for s in services.values()
        )
        any_unhealthy = any(
            s["status"] in ("unavailable", "timeout", "error")
            for s in services.values()
        )
        overall = "healthy" if all_healthy else ("unhealthy" if any_unhealthy else "degraded")

        started_at = getattr(app.state, "started_at", datetime.now(timezone.utc))
        uptime = (datetime.now(timezone.utc) - started_at).total_seconds()

        status_code = 200 if overall == "healthy" else 503
        return JSONResponse(
            status_code=status_code,
            content={
                "status": overall,
                "service": "kerala-lpg-optimizer",
                "version": app.version,
                "uptime_seconds": round(uptime, 1),
                "services": services,
            },
        )
    ```

    **4. Apply retry decorators to external service HTTP calls in main.py:**
    The retry targets are synchronous methods in core/ modules:
    - `core/geocoding/google_adapter.py` line 64: `GoogleGeocoder._call_api(self, address)` -- uses `httpx.get()`
    - `core/optimizer/vroom_adapter.py` line 103: The optimize method's `httpx.post()` call

    **Approach: Call-site wrappers in main.py** (avoids modifying core/ files which are shared infrastructure):

    In main.py, after the retry import, create two wrapper functions:
    ```python
    @geocoding_retry
    def _geocode_with_retry(geocoder: GoogleGeocoder, address: str):
        """Wrap geocoder._call_api with transient failure retry."""
        return geocoder.geocode(address)

    @optimizer_retry
    def _optimize_with_retry(optimizer: VroomAdapter, orders, vehicles):
        """Wrap optimizer.optimize with transient failure retry."""
        return optimizer.optimize(orders, vehicles)
    ```

    Then replace the direct calls:
    - In the upload_and_optimize endpoint, where `cached_geocoder` calls happen (~line 925-970), wrap the geocoding calls. Since CachedGeocoder wraps GoogleGeocoder, apply retry to the CachedGeocoder's upstream call. The simplest approach: patch `GoogleGeocoder._call_api` at the call site by wrapping it: `geocoder._call_api = geocoding_retry(geocoder._call_api)` right after `_get_geocoder()` returns the instance. This applies retry to every individual geocoding HTTP request.
    - For VroomAdapter (~line 1083), similarly wrap: `optimizer = VroomAdapter(...)` then `optimizer.optimize = optimizer_retry(optimizer.optimize)` before calling it. This retries the VROOM HTTP POST on transient failures.

    This is the cleanest approach because:
    - No modification to core/ files (keeps them dependency-free)
    - Retry applies per-request (each HTTP call retried individually, not the whole batch)
    - Both decorators are synchronous (matching the synchronous httpx calls in the adapters)
