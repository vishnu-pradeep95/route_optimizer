<objective>
Fix the address_display data flow bug and add storage for completely unprocessed CDCMS text, so the API serves cleaned original addresses instead of Google's formatted_address.

Purpose: Drivers currently see Google's formatted_address (e.g., "Vatakara, Kerala, India") instead of the actual delivery address from CDCMS. This is the core bug that makes routes show wrong locations. Fixing it requires changing the data source at two code sites, adding a new DB column for unprocessed text, and updating the API response to include both cleaned and unprocessed addresses.

Output: Fixed data pipeline where address_display always contains cleaned CDCMS text, new address_original column stores unprocessed text, and API response includes both fields. Targeted tests proving the bug is fixed and the new field is served.
</objective>
