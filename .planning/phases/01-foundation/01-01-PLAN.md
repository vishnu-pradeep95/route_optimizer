  <files>apps/kerala_delivery/dashboard/package.json, apps/kerala_delivery/dashboard/vite.config.ts</files>
  <action>
From the `apps/kerala_delivery/dashboard/` directory, install the three required packages:
```bash
npm install tailwindcss@4.2.1 @tailwindcss/vite@4.2.1 daisyui@5.5.19
```

Then update `vite.config.ts` to add the Tailwind Vite plugin BEFORE the React plugin (order matters for performance):

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [
    tailwindcss(),  // Must be before react() for optimal performance
    react(),
  ],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
      '/health': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
    },
  },
})
```

Do NOT use tailwind.config.js (that is a v3 pattern). Tailwind v4 is CSS-first -- all configuration goes in CSS via @import and @plugin directives.
  </action>

<task type="auto">
  <name>Task 2: Add Tailwind import with prefix(tw) and DaisyUI plugin to CSS entry point</name>
  <files>apps/kerala_delivery/dashboard/src/index.css</files>
  <action>
Add the Tailwind import and DaisyUI plugin directives at the TOP of `src/index.css`, BEFORE all existing styles. The existing CSS must remain completely unchanged below the new directives.

Add these three lines at the very top of `src/index.css`:

```css
/* Tailwind with prefix to prevent CSS variable collision with existing --color-* tokens */
@import "tailwindcss" prefix(tw);

/* DaisyUI component library (classes also use tw: prefix) */
@plugin "daisyui";

```

Then keep ALL existing CSS below, starting with the original comment block and the `*, *::before, *::after { box-sizing: border-box; }` rule. Do not modify, reorder, or remove any existing CSS.

The `prefix(tw)` directive is CRITICAL -- it namespaces:
- All Tailwind utility classes as `tw:flex`, `tw:bg-red-500`, etc.
- All Tailwind CSS variables as `--tw-color-*`, `--tw-font-*`, etc.
- All DaisyUI component classes as `tw:btn`, `tw:card`, `tw:table`, etc.

This prevents collision with the existing `--color-accent`, `--color-surface`, `--color-text-primary` etc. tokens on `:root`.

After editing, verify the Vite dev server starts without errors:
```bash
cd apps/kerala_delivery/dashboard && npx vite build
```

The build should complete without CSS parsing errors. If it fails, the most likely cause is a syntax error in the @import line -- ensure the exact syntax `@import "tailwindcss" prefix(tw);` is used (no colon after prefix, parentheses around tw).
  </action>
  <verify>
    <automated>cd apps/kerala_delivery/dashboard && npx vite build 2>&1 | tail -5</automated>
  </verify>
  <done>index.css starts with `@import "tailwindcss" prefix(tw);` and `@plugin "daisyui";` followed by all original CSS unchanged. `vite build` succeeds without errors. The existing --color-* variables remain in the CSS output unmodified.</done>
</task>
