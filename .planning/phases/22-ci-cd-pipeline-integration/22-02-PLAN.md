<task type="auto">
  <name>Task 1: Add E2E test job to GitHub Actions CI</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Add a 4th job `e2e` to `.github/workflows/ci.yml` with the following specification:

    ```yaml
    e2e:
      name: E2E Tests
      runs-on: ubuntu-latest
      if: github.event_name == 'push'
      needs: [test, dashboard]
    ```

    E2E runs only on push to main (not PRs). It depends on `test` and `dashboard` jobs passing first -- no point running expensive E2E if basic validation fails. Do NOT depend on `docker` job (E2E rebuilds images via compose).

    Steps in order:

    1. **Checkout code** -- `actions/checkout@v4`

    2. **Set up Node.js 22** -- `actions/setup-node@v4` with `cache: "npm"` and `cache-dependency-path: package-lock.json`

    3. **Install npm dependencies** -- `npm ci`

    4. **Install Playwright Chromium** -- `npx playwright install chromium --with-deps` (the `--with-deps` flag installs OS-level dependencies like libgbm, libasound that Chromium needs on ubuntu-latest)

    5. **Start Docker Compose stack** -- `docker compose up -d --wait --wait-timeout 180`. The `--wait` flag blocks until all services with healthchecks are healthy (db, osrm, api). Use 180s timeout because osrm-init downloads ~150 MB on first run. Environment variables:
       - `API_KEY: ${{ secrets.API_KEY }}`
       - `ENVIRONMENT: development`
       Include the FULL stack (all services including osrm-init, osrm, vroom). The E2E tests call `uploadTestCSV()` which triggers VROOM optimization -- without VROOM, 30+ of 38 tests fail. Despite the success criteria mentioning "without OSRM/VROOM", the tests cannot pass without them.

    6. **Run Playwright E2E tests** -- `npx playwright test --workers=1`. Environment:
       - `API_KEY: ${{ secrets.API_KEY }}`
       - `CI: true`

    7. **Capture Docker logs on failure** -- `if: failure()`, run `docker compose logs --no-color > docker-logs.txt 2>&1`. Also capture license-test container logs if it exists: `docker compose -f docker-compose.yml -f docker-compose.license-test.yml logs api-license-test >> docker-logs.txt 2>&1 || true`

    8. **Upload Playwright report on failure** -- `if: failure()`, use `actions/upload-artifact@v4`:
       - name: `playwright-report`
       - path: `playwright-report/`
       - retention-days: 7

    9. **Upload Docker logs on failure** -- `if: failure()`, use `actions/upload-artifact@v4`:
       - name: `docker-logs`
       - path: `docker-logs.txt`
       - retention-days: 7

    10. **Tear down Docker Compose** -- `if: always()`, run `docker compose -f docker-compose.yml -f docker-compose.license-test.yml down -v` (clean up both main stack and license-test container, remove volumes to not pollute runner disk)

    IMPORTANT constraints:
    - Use `actions/upload-artifact@v4` (NOT v3 which is deprecated)
    - Artifact upload steps use `if: failure()` (NOT `if: always()` -- per user decision, upload ONLY on failure)
    - Keep the existing 3 jobs (test, dashboard, docker) completely unchanged
    - Do not change the workflow `name`, `on` triggers, or `concurrency` settings
  </action>

    <automated>grep -c "E2E Tests" .github/workflows/ci.yml && grep -c "playwright" .github/workflows/ci.yml && grep -c "upload-artifact" .github/workflows/ci.yml && grep -c "failure()" .github/workflows/ci.yml</automated>
  </verify>
  <done>CI workflow has 4 jobs: Python Tests, Dashboard Build, Docker Build, and E2E Tests. E2E job starts full Docker Compose stack, installs Chromium, runs Playwright, uploads artifacts on failure, and tears down.</done>
