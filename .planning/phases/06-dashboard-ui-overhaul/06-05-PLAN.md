Modify the `get_qr_sheet` endpoint's inline CSS and HTML template in `apps/kerala_delivery/api/main.py`:

1. **Increase QR image size** from 150px to 210px:
   Update the `.qr-img` CSS:
   ```css
   .qr-img {{
     width: 210px;
     height: 210px;
   }}
   ```
   Why 210px: Optimizes for reliable scanning from ~30cm (arm's length in a Piaggio Ape three-wheeler cab). 200-220px is the recommended range from the research; 210px is the midpoint.

2. **Increase QR PNG resolution:** Change the `generate_qr_base64_png` call's `box_size` from 6 to 8:
   ```python
   "qr_pngs": [
       generate_qr_base64_png(seg["url"], box_size=8)
       for seg in segments
   ],
   ```
   Why: Larger display size needs higher resolution PNG to avoid pixelation when printed at 300dpi.

3. **Add modern break-inside property** alongside the existing legacy property:
   ```css
   .card {{
     border: 2px solid #333;
     border-radius: 8px;
     padding: 16px;                    /* Increased from 12px for more breathing room */
     break-inside: avoid;              /* Modern CSS Fragmentation Level 3 */
     page-break-inside: avoid;         /* Legacy fallback */
   }}
   ```

4. **Increase card header font sizes** for arm-length readability:
   ```css
   .vehicle-id {{
     font-size: 22px;                  /* Increased from 18px */
     font-weight: 800;
     font-family: 'Courier New', monospace;
   }}
   .driver-name {{
     font-size: 16px;                  /* Increased from 14px */
     color: #333;                      /* Darker from #444 for better print contrast */
   }}
   ```

5. **Add tabular-nums to card stats** for numeric alignment:
   ```css
   .card-stats {{
     display: flex;
     gap: 16px;                        /* Increased from 12px */

   }}
   ```

6. **Increase scan instruction font size:**
