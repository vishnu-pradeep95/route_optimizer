**2. Debug logging gate (PWA-06):**

In `index.html`, at the very top of the `<script>` block (before `API_BASE` declaration, around line 872), add:

```javascript
// Debug mode: append ?debug=1 to URL to enable console logging.
// Without it, console.log is silenced for clean production output.
// console.warn is always active — those indicate real problems.
const DEBUG = new URLSearchParams(window.location.search).has('debug');
if (!DEBUG) {
    console.log = () => {};
}
```

This approach:
- Checks for `?debug=1` (or any `?debug` param) in the URL
- Overrides `console.log` with a no-op when debug is off
- Preserves `console.warn` and `console.error` always (real errors must be visible)
- Requires zero changes to existing log call sites -- they all just work silently
- Office staff can tell drivers to add `?debug=1` to the URL for field debugging (per user decision)

**Do NOT gate SW logs** in `sw.js` -- the service worker runs in a separate scope and its 3 console calls are infrastructure-level (install/activate). They fire rarely and are useful for diagnosing SW issues. Leave them as-is.
