Create scripts/start.sh with these sections, following the exact color helper pattern from bootstrap.sh:

**Header comment block** (match bootstrap.sh style):
- Script title: "Daily Startup — Kerala LPG Delivery Route Optimizer"
- What it does: starts Docker daemon if needed, runs compose up, polls health, prints URLs
- Usage: `cd ~/routing_opt && ./scripts/start.sh`
- Duration: ~10-30 seconds (services already built)

**Shell options:** `set -euo pipefail`

**Color helpers:** Copy verbatim from bootstrap.sh lines 33-48 (the `if [ -t 1 ]` block plus info/success/warn/error/header functions). Do NOT invent new helper names.

**Constants:**
```bash
HEALTH_URL="http://localhost:8000/health"
MAX_WAIT=60
POLL_INTERVAL=3
```

**Project root navigation:** Use `SCRIPT_DIR` pattern from bootstrap.sh to cd to the project root (parent of scripts/):
```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
cd "$PROJECT_ROOT"
```

**Function: ensure_docker_running()**
- Check `docker info &>/dev/null` -- if true, return 0 silently (no output for the happy path when Docker is already running)
- If not running, print `info "Starting Docker..."` and run `sudo service docker start &>/dev/null`
- Wait up to 10 seconds for daemon to be ready (loop checking `docker info` with `sleep 1`)
- On success: `success "Docker started"`
- On failure: `error "Could not start Docker."` then print two suggestions: (1) `sudo service docker start` (2) restart WSL via PowerShell `wsl --shutdown`. Return 1.
- IMPORTANT: The `docker info` check must use `if` or `||` to avoid `set -e` exit on non-zero

**Function: poll_health()**
- Loop from 0 to MAX_WAIT in POLL_INTERVAL increments
- Each iteration: try `curl -sf "$HEALTH_URL" >/dev/null 2>&1`, return 0 on success
- Print spinner progress: use the same 4-frame spinner from install.sh (lines 271-276): characters `⠋ ⠙ ⠹ ⠸`
- Format: `\r  {spinner} Waiting for services... ({elapsed}s / {MAX_WAIT}s)  `
- After loop exhausts, print newline and return 1

**Function: diagnose_failure()**
- Print `error "System did not become healthy within ${MAX_WAIT} seconds."`
- Check three critical services using `docker inspect --format='{{.State.Status}}'`:
  - `lpg-db` (Database) -- also check `{{.State.Health.Status}}` since it has a healthcheck
  - `osrm-kerala` (OSRM routing engine) -- check `.State.Status` only (OSRM healthcheck is unreliable per research Pitfall 4)
  - `lpg-api` (API server) -- check `.State.Status`
- For each non-running/non-healthy service, print: `error "{Service name} ({container}): {status}"` followed by `echo "  Try: docker compose logs {service} --tail=20"`
- If ALL containers appear running but API doesn't respond, print: `warn "All containers appear running but API is not responding."` with suggestion to wait and retry `curl http://localhost:8000/health`
- Print a final catch-all: `echo "  If the problem persists, try: docker compose down && ./scripts/install.sh"`

**Main flow (no function, just sequential):**
1. Print a startup header: `header "Kerala LPG Delivery Route Optimizer"` then `info "Starting daily services..."`
2. Call `ensure_docker_running || exit 1`
3. Print `info "Starting containers..."` then run `docker compose up -d` (NO `--build` flag -- daily startup must NOT rebuild images)
4. Suppress the docker compose output noise: redirect stdout to /dev/null but keep stderr: `docker compose up -d > /dev/null 2>&1`. Actually, let compose output flow -- it's informative ("Container lpg-api Running") and completes in <2 seconds.
   CORRECTION: Use `docker compose up -d 2>&1 | while IFS= read -r line; do echo "  $line"; done` to indent compose output, OR simply run `docker compose up -d` and let output flow naturally. Simplest approach: just run `docker compose up -d` with no redirection.
5. Print blank line, then call `poll_health`
6. On success (poll_health returns 0):
   - Print success banner matching install.sh style:
     ```
     (green)═══════════════════════════════════════════════════════════════(reset)
     (green)  ✓ System is running!                                        (reset)
     (green)═══════════════════════════════════════════════════════════════(reset)

       Dashboard:   http://localhost:8000/dashboard/
       Driver App:  http://localhost:8000/driver/
     ```
   - Exit 0
7. On failure (poll_health returns 1):
   - Call `diagnose_failure`
   - Exit 1

**Make executable:** `chmod +x scripts/start.sh`

**What to AVOID and WHY:**
- Do NOT use `--build` flag on docker compose up (wastes time, rebuilds images every morning)
- Do NOT use `docker compose down` before up (destroys init container completion state, causes 10+ minute OSRM re-processing)
- Do NOT check individual services before compose up (compose handles all orchestration)
- Do NOT use `docker-compose` with hyphen (v1 deprecated, project uses v2 plugin `docker compose`)
- Do NOT use 300-second timeout (that's for install.sh first-time setup; daily startup uses 60 seconds)
- Do NOT add any interactive prompts (read, select, etc.) -- zero-input is a hard requirement
