  <name>Task 1: Add rapidfuzz dependency and create dictionary build script</name>
  <files>requirements.txt, scripts/build_place_dictionary.py</files>
  <action>
1. Add `rapidfuzz==3.14.3` to `requirements.txt` (alphabetically sorted — between `qrcode` and `requests`). Run `pip install rapidfuzz==3.14.3` to install it locally.

2. Create `scripts/build_place_dictionary.py` — a standalone script that generates `data/place_names_vatakara.json` by merging three data sources:

**Script structure:**
- Constants: DEPOT_LAT=11.6244, DEPOT_LON=75.5796, RADIUS_M=30000, OUTPUT_PATH=data/place_names_vatakara.json
- `fetch_osm_places()`: POST to https://overpass-api.de/api/interpreter with the Overpass query from research. Extract name, name:ml, lat, lon, place type for each element. Uppercase the name. Return list of dicts.
- `fetch_india_post_offices()`: GET https://api.postalpincode.in/pincode/{pin} for each PIN in 673101-673106. Extract Name (uppercased), Pincode, District. NOTE: no lat/lon available from this API. Return list of dicts.
- `get_manual_seeds()`: Return the hardcoded list of CDCMS-specific names not in either API (from research MANUAL_SEEDS). These are critical for CDCMS coverage.
- `merge_and_deduplicate(osm, india_post, seeds)`: Merge all three sources. Use RapidFuzz `fuzz.ratio` at 85% threshold to detect duplicates across sources. For each entry, prefer OSM coordinates when available. For India Post entries without OSM match, set `coordinates_approximate: true` with depot coordinates as placeholder. Sort alphabetically by name.
- `validate_coverage(entries, sample_csv_path)`: Read `data/sample_cdcms_export.csv`, extract distinct AreaName values, check what percentage are covered by the dictionary (exact or fuzzy match at 85%). Print coverage report. Return coverage percentage.
- `main()`: Parse args (--dry-run flag for testing), call all functions, write JSON, run coverage validation, exit with error if coverage < 80%.

**Output JSON schema:**
```json
{
  "metadata": {
    "generated": "2026-03-11T...",
    "depot": {"lat": 11.6244, "lon": 75.5796},
    "radius_km": 30,
    "sources": ["osm_overpass", "india_post", "manual_seeds"],
    "entry_count": 285
  },
  "entries": [
    {
      "name": "MUTTUNGAL",
      "aliases": ["MUTUNGAL"],
      "type": "village",
      "source": "india_post",
      "lat": 11.6300,
      "lon": 75.5800,
      "pincode": "673106",
      "name_ml": null,
      "coordinates_approximate": false
    }
  ]
}
```

**Important implementation details:**
- Use `requests` for HTTP calls (already in requirements.txt)
- Use `rapidfuzz.fuzz.ratio` for deduplication matching
- Add retry logic (3 attempts with 2s sleep) for API calls since Overpass can be slow
- Add `--dry-run` flag that prints what would be fetched but does not call APIs (reads from cached JSON if exists)
- Script must be idempotent — re-running produces equivalent output
- Add proper error handling for API failures (log warning, continue with available data)
- Use `if __name__ == "__main__"` guard
- Include compound names like "CHORODE EAST", "MUTTUNGAL WEST" as entries (critical for longest-match splitting)
