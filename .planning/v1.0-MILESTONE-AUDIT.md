## Integration Check

**Score:** 14/16 exports properly wired, 2 with degraded edge cases

### Issue 1: fetchHealth 503 Handling (ERR-05, ERR-09)

`/health` returns HTTP 503 with per-service JSON body on degraded state. `apiFetch()` throws on non-2xx, causing `App.tsx` to lose the per-service data and display only "Unhealthy" instead of per-service breakdown.

**Impact:** Health bar loses granularity precisely when it's most needed. Non-blocking — works correctly when all services are healthy.

### Issue 2: ERROR_HELP_URLS Broken Anchors (ERR-01)

All 15 help URL anchor fragments in `errors.py` and `errors.ts` don't match actual markdown heading anchors in target docs. Users navigate to the correct doc file but land at the top instead of the relevant section.

**Impact:** Degraded help navigation, not broken. Files resolve correctly; only anchors are wrong.

### Issue 3: ERROR-MAP.md Stale Line Numbers (DOC-VALIDATE)

Phase 02 added ~140 lines to main.py, shifting all line references in `docs/ERROR-MAP.md`.

**Impact:** Developer documentation accuracy only.

### Issue 4: FleetManagement.tsx Stale Comments (DOC-CLEANUP)

Code comments at lines 53, 74 reference deleted `plan/` directory. Phase 01 scoped cleanup to .github/ and docs, not source code comments.

**Impact:** Code comments only. No runtime impact.
